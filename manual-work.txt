- これがないとFirebase Admin SDKがiOSデバイスに通知を送信できない

**APNs認証キーの取得方法（未設定の場合）**:
1. Apple Developer Portal (https://developer.apple.com/account/resources/authkeys/list) にアクセス
2. 「Keys」から「+」ボタンをクリック
3. Key Name を入力（例: "Okiroya FCM Key"）
4. 「Apple Push Notifications service (APNs)」にチェック
5. 「Continue」→「Register」をクリック
6. .p8ファイルをダウンロード（**一度しかダウンロードできない**）
7. Key ID をメモ

**Firebase Consoleへの登録**:
1. Firebase Console > プロジェクト設定 > Cloud Messaging
2. 「Apple アプリの構成」で「APNs認証キー」の「アップロード」をクリック
3. .p8ファイルをアップロード
4. Key ID を入力
5. Team ID を入力（Apple Developer PortalのMembership詳細で確認）

### Work 2.15.4: testAlarm vs checkAlarms の比較テスト [完了]

- [x] サイレントスイッチ: オフ、音量: 0ではない ことを確認

### 調査結果
**根本原因**: `setBackgroundMessageHandler`が登録されている場合、iOSでFCM通知の音声が再生されない
**解決策**: バックグラウンドハンドラー内でNotifeeの`displayNotification()`を使用して明示的に音声付き通知を表示する
**参考**:
- https://github.com/invertase/react-native-firebase/discussions/4633
- https://github.com/invertase/react-native-firebase/issues/2634
