# プロジェクトルール

このプロジェクトでは以下のルールを**必ず**遵守すること。

---

## 絶対遵守ルール

### 1. スクワット検知の信頼性
**「ユーザーがスクワットをしたにも関わらずスクワットを認知できない」という状況は絶対に起こらないアプリにする**

- False Negative（見逃し）は絶対NG
- 判定閾値は常にキャリブレーションデータより緩く設定
- 疑わしい動作は「スクワットとして認める」方向で判定
- テスト時は見逃しケースを重点的に検証

### 2. UI実装時のスキル使用
**UIを実装する際は絶対に`frontend-design`スキルを使用する**

- コンポーネント作成時は必ず`/example-skills:frontend-design`を実行
- デザイン品質を担保するため例外なく適用

### 3. 実装範囲の厳守
**各タスクを実行しアプリの開発をする際は絶対に指示された範囲外の実装はしない**

- 指示されたタスクのみを実装
- 「ついでに」「改善として」などの追加実装は禁止
- 範囲外の改善点に気づいた場合は報告のみ行い、実装はしない

### 4. Phase単位での作業と確認
**タスクを実行する際はひとつのPhase毎に実行し、そのPhaseのタスクが全て終了したら一度作業を止めユーザーの次の指示を仰ぐ**

- 1つのPhaseが完了したら必ず停止
- 次のPhaseに自動的に進まない
- 完了報告を行い、ユーザーの指示を待つ

### 5. タスク追加時の配置ルール
**タスクを追加する場合はテストのPhaseが全タスクの1番最後になるようにしつつ、全てのタスクを円滑に消化できる箇所にタスクを追加する**

- 正しく優先順位・依存関係を考慮して配置
- 新しく追加したタスクを安直にタスク一覧の1番最後に追加しない
- テストPhase → ビルド・リリースPhase の順序は常に維持

### 6. アラーム音設定機能について
**ユーザーがアラーム音を選択・アップロードする機能は実装しない**

- アラーム音はアプリにバンドルされたデフォルト音（アラーム音.m4a / alarm.caf）のみ使用
- カスタムサウンド選択UI、音源アップロード機能は実装しない
- コード内の`customAlarmSound`関連の引数は将来の拡張用として残すが、UIは提供しない

### 7. 問題発見時のワークフロー
**アプリの問題（バグ・不具合）が発見された場合は、必ず以下の手順で対応する**

1. **原因調査**: 丁寧にコードを調査し、問題の原因を特定する
2. **problem.txtに記録**: 問題の詳細（症状、原因、影響範囲）を`problem.txt`に追加
   - フォーマット: `## Problem N: [問題タイトル] (YYYY-MM-DD)`
3. **修正計画を立案**: 解決に必要なタスクを洗い出す
4. **タスク追加**:
   - AI実装タスク → `progress.txt`に追加
   - 手動確認・設定タスク → `manual-work.txt`に追加
5. **依存関係を考慮**: テストPhase・ビルドPhaseの前に配置

### 8. 手動作業の発見時は必ずmanual-work.txtに追加
**作業中に手動で行う必要がある作業（API有効化、Console設定、外部サービス設定など）を発見した場合は、必ず即座に`manual-work.txt`に追加する**

- 発見した時点で即座に追加（後回しにしない）
- 具体的な手順・URLを含める
- 既存のWorkに関連する場合は該当Workに追記
- 新規の場合は新しいWork番号を付与
- 例: API有効化、IAMロール設定、Firebase Console設定、外部サービス連携設定など

## タスク管理

- `progress.txt`: AI実装タスク（Phase単位）
- `manual-work.txt`: 手動作業タスク（Work単位）
- `archive.txt`: 完了済みタスク

完了したタスクはチェックマーク `[x]` を付け、Phase/Work全体完了時に`archive.txt`へ移動する。

## 技術スタック

- フロントエンド: React Native (Expo)
- 認証: Firebase Authentication
- データベース: Firestore
- サーバーレス: Firebase Functions
- 通知: Firebase Cloud Messaging
- センサー: Core Motion（加速度センサー）
- SNS連携: X API v2（Instagram Graph APIは将来対応）
- 課金: RevenueCat または expo-in-app-purchases
- 多言語化: react-i18next / expo-localization（日本語・英語対応）
